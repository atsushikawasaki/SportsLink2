# SportsLink UI/UX 改善ガイドライン

SportsLinkを「スポーツを通じた最高のコミュニティ体験」を提供するプラットフォームにするための、UI（インターフェース）およびUX（ユーザー体験）の設計指針を定義します。

## 1. デザインの核となる3原則 (Core UI Principles)

### ① 明確な視覚的階層 (Visual Hierarchy)

ユーザーが画面を見た瞬間に「何が最も重要か」を理解できる必要があります。

- **アクションの強調**: 「参加する」「予約確定」などのメインアクション（CTA）は、アクセントカラーと適切なサイズを用いて視覚的に最優先します。
- **情報の整理**: Tailwind CSSの z-index や shadow を活用し、カード型レイアウトなどで情報の優先順位を明確にします。

### ② 徹底した一貫性 (Consistency)

アプリ全体で操作ルールを統一し、ユーザーの学習コストを最小化します。

- **共通パーツの使用**: ボタン、入力フォーム、モーダル、トースト通知などのUIコンポーネントを共通化します。
- **デザイントークンの遵守**: 余白（Spacing）、角丸（Border Radius）、カラーパレットのルールを厳守します。

### ③ リアルタイム・フィードバック (Feedback)

ユーザーの操作に対し、システムが即座に応答を返します。

- **インタラクション**: ボタン押下時の色変化、ホバーエフェクト、ローディング中のスケルトンスクリーンを必ず実装します。
- **通知**: 成功・エラー時にはトーストメッセージを表示し、次のアクションを促します。

## 2. 優れたユーザー体験の条件 (Core UX Principles)

### ① 摩擦のない導線 (Frictionless Experience)

目的達成（イベント参加・主催）までのステップを最小限にします。

- **クイックアクセス**: 検索フィルタには「今日開催」「初心者歓迎」などのクイックタグを設置し、タップ回数を減らします。
- **フォームの簡略化**: 入力項目を精査し、自動補完や選択式を活用して入力のストレスを排除します。

### ② 信頼性と透明性 (Trust & Transparency)

対面での活動を扱うため、ユーザーが安心できる情報を提示します。

- **情報の可視化**: 主催者の評価、過去の活動実績、参加予定者のレベル感を明示します。
- **キャンセルポリシー**: 支払い前や参加確定前に、規約を分かりやすく提示します。

### ③ モバイルファースト設計

スポーツの現場（コート）での片手操作を最優先します。

- **親指ゾーン**: 重要なボタンは画面下部や親指が届きやすい範囲に配置します（Bottom Navigationの採用）。
- **タップターゲット**: すべてのクリック要素は 48x48px 以上のサイズを確保します。

## 3. 改善プロセス

開発を進める際は、以下のステップを推奨します。

1. **コンポーネントの共通化**
   - 「このプロジェクトで使われているボタンのバリエーションを分析し、共通の Button.tsx に統合して」と指示する。

2. **アクセシビリティ監査**
   - 「このページのコントラスト比を確認し、屋外で見にくい箇所を修正して」と依頼する。

3. **マイクロインタラクションの実装**
   - 「framer-motion を使って、カードがふわっと表示されるアニメーションを追加して」と指示し、質感向上を図る。

## 4. 評価指標 (KGI/KPI)

改善の効果を以下の数値で測定します。

- **コンバージョン率 (CVR)**: イベント詳細閲覧から「参加申し込み」に至る割合。
- **リピート率**: 30日以内に再度イベントに参加または主催したユーザーの割合。
- **離脱率**: 検索画面から詳細を見ずに離脱するユーザーの割合。
